# MixtarRVS P0

Status: zaliczone.

P0 potwierdza powtarzalny lancuch `Linux -> initramfs -> OpenRC PID1 -> ash/zsh -> kontrolowany poweroff/reboot` uruchamiany bezposrednio przez QEMU. Nie jest to jeszcze autonomiczny obraz GPT/UEFI ani VHDX. Ten zakres nalezy do P1.

## Zwykly build

~~~powershell
powershell -NoProfile -ExecutionPolicy Bypass -File Scripts/build-firstboot.ps1
~~~

Polecenie waliduje `Layout.config`, buduje komponenty, publikuje initramfs i wykonuje dwa oddzielne testy QEMU.

## Build od pustego cache

Nowy namespace musi nie istniec przed uruchomieniem:

~~~powershell
$namespace = "p0-cold-$([guid]::NewGuid().ToString('N'))"
powershell -NoProfile -ExecutionPolicy Bypass -File Scripts/build-firstboot.ps1 -CacheNamespace $namespace
~~~

Gdy wolumin repozytorium nie ma wystarczajacej ilosci miejsca, mozna jawnie podac baze cache hosta:

~~~powershell
powershell -NoProfile -ExecutionPolicy Bypass -File Scripts/build-firstboot.ps1 -CacheNamespace $namespace -HostCacheBase "<wolumin>:\MixtarRVS-P0-Cold"
~~~

`CacheNamespace` izoluje jednoczesnie cache zrodel po stronie Windows oraz `XDG_CACHE_HOME` po stronie WSL. Domyslne cache nie jest wtedy modyfikowane.

Ostatni zaliczony przebieg zakonczyl sie: `2026-07-15T17:27:07.8373294Z`.

## Dowody

- `Output/P0/Build.json`: wspolny manifest wersji, URL-i, commitow, patchy, wejsc i SHA-256.
- `Output/P0/Contract.json`: wynik walidacji kontraktu nazw, hosta i publicznego FHS.
- `Output/P0/Qemu-firstboot.json`: trwaly raport kontrolowanego poweroff.
- `Output/P0/Qemu-reboot.json`: trwaly raport kontrolowanego rebootu.
- `Output/P0/Qemu-firstboot.log` i `Output/P0/Qemu-reboot.log`: surowe logi konsoli szeregowej.
- `Output/P0/ColdBuild.json`: raport pelnego przebiegu od nieistniejacego namespace cache.
- `Output/P0/ColdBuild.log`: polaczony zapis stdout i stderr cold builda.
- `Output/P0/MixtarRVS-firstboot.cpio.gz`: przetestowany initramfs.

Cold build jest zaliczony tylko wtedy, gdy log zawiera `P0_CONTRACT_OK`, `"cached": false`, `FIRST_BOOT_OK`, `REBOOT_OK` oraz `FIRST_BOOT_PIPELINE_OK`.
